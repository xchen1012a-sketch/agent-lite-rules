# Skill 执行契约

所有 `skills/*/SKILL.md` 命中后都必须遵守本契约。

## 强制规则

- 命中 skill 后必须读取对应 `SKILL.md`，不能只看索引。
- 先读取最小上下文，再改文件、跑命令或给结论。
- 按 skill 的工作流、停止条件和验证方式执行。
- 写代码保持最小必要改动；不做未要求的扩展、抽象、重构或格式化大改。
- 非 L0 写代码任务先明确假设、成功标准和验证方式；不确定时先报告，不静默猜。
- 项目 `.ai-spec/skills/` 优先；本地 skill 只能补充，不能覆盖项目红线、契约、任务等级和验证要求。
- 涉及真实外部服务、生产数据、权限、安全、DB、文件删除、Git push、全局配置时，先取得用户确认。
- 不写入或修改用户全局 Claude / Codex 配置、全局 rules、全局 skills、全局 hooks、全局 MCP。
- 输出保持模块化，说明使用的 skill、变更、验证、未验证项和是否触发停止条件。

## 冲突处理

同级规则冲突时，以当前项目 `.ai-spec/`、用户当前明确要求和更具体的 skill 规则为准。
