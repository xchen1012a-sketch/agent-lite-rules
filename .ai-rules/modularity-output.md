# 模块化代码与输出

本规则约束 AI 写代码、写文档、写报告和交付说明的组织方式。

## 模块化代码

代码必须按职责拆分，不把多个层级、多个目的、多个变化原因混在同一文件或同一函数里。

- 单一职责：一个模块只负责一类清晰职责。
- 边界清楚：UI、API client、service、repository、worker、schema、config、test 不混写。
- 入口轻：入口文件只做组装和路由，不堆业务细节。
- 复用克制：真实重复出现后再抽象，不为预感创建大框架。
- 依赖单向：上层调用下层，禁止循环依赖和隐式跨层访问。
- 可测试：核心逻辑必须能脱离 UI、进程和真实外部服务测试。
- 可替换：外部服务、LLM、媒体、DB、队列必须有边界封装，不在业务代码里散落调用。

## 简洁实现

默认选择能满足需求的最小实现。

- 不写用户没要求的功能、配置项、兼容层或“未来扩展”。
- 不为一次性代码创建抽象；真实重复出现后再抽象。
- 不用 100 行框架解决 10 行代码能解决的问题。
- 每一处改动都应能对应用户需求、修复证据或验证需要。
- 匹配现有项目风格；不顺手格式化、重命名或重构无关代码。
- 只清理本次改动造成的无用 import、变量、函数；历史遗留问题先报告，不擅自删除。

## 动手前确认

非 L0 写代码任务，动手前先明确成功标准。

- 需求有多种解释时，先说明假设和取舍；影响范围不同的，先问用户。
- 成功标准必须可验证：bugfix 要有复现/修复/复验，功能要有输入/输出/验收方式。
- 涉及框架、UI 库、ORM、SDK API 时，先确认当前项目版本、类型定义、本地文档或官方用法，不凭记忆硬写。
- 如果最小实现会改变 API、DB、权限、生成目录或禁改目录，先停下来报告影响。

## 命名规范

文件、目录、模块、类、函数、组件、API、阶段编号必须来自项目语义，不能由 AI 随意发挥。

命名来源优先级：

1. 现有项目约定：已有目录、文件、模块、领域词、命名风格。
2. 项目事实：`.ai-rules/project-facts.md` 中的项目名、业务域、模块词表。
3. 已确认计划：`docs/plans/project-plan.md` 和当前 phase 中确认过的模块名。
4. 用户当前明确指定的名称。

- 同一概念只用一个名称，不混用同义词。
- 模块名表达业务职责，不使用 `common2`、`newService`、`managerHelper`、`tempModule` 这类模糊名称。
- 文件名与主要导出、组件、service 或 schema 对齐。
- 阶段文件名使用 `<PHASE-ID>-<kebab-name>.md`。
- 新增目录前先判断是否已有同类目录，避免重复创建 `utils`、`services`、`components` 的平行副本。

禁止：

- 一个文件同时承担 UI、请求、业务、持久化、配置。
- handler/controller 直接堆业务、SQL、外部服务调用。
- 组件里直接散落 API 字段猜测和复杂状态转换。
- 为了省事新增“万能工具类”“万能 store”“万能 service”。
- 为了赶进度把 mock、临时开关、调试逻辑混进正式路径。
- 在同一项目中混用中文拼音、英文、缩写和随意简称。
- 用不表达业务职责的泛名创建长期模块。
- 无法确定命名时直接新建；必须先报告候选名称和理由。

## 模块化输出

AI 的输出也必须模块化，让用户能快速确认范围、风险和下一步。

默认输出结构：

1. 结论：当前判断或结果。
2. 变更：改了哪些模块或文件。
3. 验证：运行了什么，结果是什么。
4. 风险：没验证什么，剩余风险是什么。
5. 下一步：需要用户确认或后续处理的事项。

输出要求：

- 不把计划、解释、结果、风险混成一大段。
- 不用“应该没问题”代替验证结果。
- 涉及计划外文件时，单独列出“计划外文件报告”。
- 涉及多仓时，按仓库分别输出。
- 涉及多个模块时，按模块分别输出。
- 涉及失败时，按“现象、复现、根因、修复、复验”输出。

## 停止条件

遇到以下情况必须停下来报告：

- 当前改动需要把多个职责硬塞进一个模块。
- 为了完成任务必须创建大而泛的抽象。
- 无法判断文件应该归属哪个模块。
- 无法确定项目名、模块名或文件命名来源。
- 输出无法清楚拆分结论、变更、验证、风险。
- 用户要求的范围会破坏现有模块边界。

## 验证方式

- 检查 diff 中是否有无关模块混入。
- 检查新增文件名、目录、函数名是否表达单一职责。
- 检查命名是否来自项目事实、现有约定或已确认计划。
- 检查核心逻辑是否有相邻测试或可验证入口。
- 检查最终回复是否按模块化输出说明真实结果。
