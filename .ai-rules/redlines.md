# 红线

## 项目级限定

本模板只能作为当前项目或当前子仓的项目级规则使用。

- 只允许写入当前项目目录内的 `CLAUDE.md`、`AGENTS.md`、`.ai-spec/`、项目内 `docs/`、项目内 `scripts/`。
- 不允许写入或修改用户本地全局配置目录，包括 `~/.claude/`、`~/.codex/`、`%USERPROFILE%\.claude/`、`%USERPROFILE%\.codex/`。
- 不允许把本模板安装成全局规则、全局 skill、全局 hook、全局 MCP 配置或全局命令。
- 不允许自动复制项目 skill 到用户本地，也不允许自动复制用户本地 skill 到项目。
- 读取用户本地 skill 只能作为按需补充，不能修改本地 skill 文件。
- 多子仓项目中，每个子仓独立接入项目级入口；父目录只做协调，除非用户明确要求父目录也成为项目。

## 通用红线

这些规则不可突破：

- 不越权读写工作区外文件。
- 不泄露密钥、Token、私钥、证书。
- 不跳过必要验证。
- 不直接执行 LLM 生成的 shell 命令。
- 不在未确认前连接真实生产数据库、支付、云服务、外部 AI 服务。
- 不自动 git commit / push，除非用户明确要求。
- 不把临时 mock、调试开关、测试密钥伪装成生产实现。
- 不为了小任务全量读取所有规则和所有文档。
- 不删除、覆盖或重置用户已有内容，除非用户明确要求。
- 不擅自修改计划外文件；先报告影响，等待用户确认。
- Claude Code 和 Codex 适配文件只能作为当前项目级入口存在。

## Git 红线

- 不自动 `git add` / `git commit` / `git push`，除非用户明确要求。
- 多 Git 子仓项目中，不在父目录提交子仓改动。
- 不提交密钥、Token、私钥、证书或真实生产配置。
