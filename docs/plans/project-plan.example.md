# 项目总计划示例

## 目标

- 建立一个可分阶段交付的项目。

## 范围

- 前端基础界面。
- 后端 API 基线。
- 本地验证脚本。

## 不做

- 不接生产数据库。
- 不接真实外部付费服务。

## 仓库边界

- 单仓：所有代码在当前项目。
- 多仓：前端、后端分别维护计划和验收。

## 技术路线

- 前端：
- 后端：
- 数据库：
- 部署：

## 阶段列表

| 阶段 | 目标 | 验收 |
|---|---|---|
| FE-01 | 前端 MVP UI | 主页面可打开，核心状态可见 |
| BE-01 | 后端 API 基线 | health/API smoke 通过 |
| OPS-01 | 本地启动脚本 | check/test/smoke 可执行 |

## 总体验收标准

- 当前阶段有明确验收证据。
- 未执行验证不会写成通过。
- 计划外文件先报告，等用户确认。

## 风险与确认项

- 单仓还是多仓。
- 是否需要真实数据库。
- 是否需要接入外部服务。
